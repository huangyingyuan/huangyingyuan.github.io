API Mock：

定好API文档之后，前后端就可以进行开发了

那么这个时候，后端的接口还没做出来，前端没有数据怎么办。这时候就是用到我们的API 智能Mock的功能了

点击刚才定义好的接口文档后，里面有个URL，我们用浏览器打开这个URL

浏览器打开后就有相应的数据了，每次点击刷新数据还不一样（非常好用）

有时候Mock出来的数据不符合我们的需求的话，我们应该如何去修改规范呢？

解答：在接口文档里面，解除关联后，点击Mock，可以选择你想要的需求去约束即可

API调试：

后端在开发的过程中，一定要把接口跑通，改接口然后自测

魔法棒：

我们可以在接口文档里面切换到 “运行” 界面，看到这里有个魔法棒（在魔法棒里我们可以自己设置不同的值的约束等）

数据结构校验：

在跑的时候，如果前后端的数据不一样的话，这里就会返回数据接口定义不一样，然后返回“校验失败”
